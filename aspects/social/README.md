# Social — Systemic Concept

Social is one of the **five systemic concepts** in OpenPersona's 4+5+3 architecture. It governs a persona's external identity and discoverability across networks and protocols.

## Role

Social bridges Soul identity data into Body-layer artifacts:
- **Soul**: identity fields (`slug`, `personaName`, `bio`, `role`) provide the source data
- **Body**: generates `agent-card.json` (A2A protocol), `acn-config.json` (ACN registry), `onchain` identity (ERC-8004)

## Declaration

Declared in `persona.json` under the top-level `social` field:

```json
{
  "social": {
    "acn": {
      "gateway": "https://acn.zeroclaw.ai",
      "tags": ["companion", "voice"]
    },
    "onchain": {
      "chain": "base",
      "identity_contract": "0x...",
      "registration_script": "scripts/register-identity.sh"
    },
    "a2a": {
      "protocol_version": "0.3.0",
      "capabilities": ["conversation", "voice"]
    }
  }
}
```

## Implementation Files

Social's implementation is entirely generator-inline — no standalone source files exist yet:

| Output File | Generated by | Role |
|-------------|-------------|------|
| `agent-card.json` | `lib/generator/social.js` | A2A agent discovery card |
| `acn-config.json` | `lib/generator/social.js` | ACN registry config |
| `wallet_address` | `lib/utils.js` `deriveWalletAddress()` | Deterministic EVM address (SHA-256 of slug) |

## Future

When Social has standalone templates (e.g. `agent-card.template.json`, `acn-config.template.json`), they will be moved here from the generator's inline logic.
