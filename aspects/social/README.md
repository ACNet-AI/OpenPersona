# Social â€” Systemic Concept

Social is one of the **five systemic concepts** in OpenPersona's 4+5+3 architecture. It governs a persona's external identity, discoverability, and relationships across networks and protocols.

## Role

Social bridges Soul identity data into Body-layer artifacts:
- **Soul**: identity fields (`slug`, `personaName`, `bio`, `role`) provide the source data
- **Body**: generates `agent-card.json` (A2A protocol), `acn-config.json` (ACN registry), `onchain` identity (ERC-8004)
- **Contact Book**: runtime contact database (`social/contacts.json`) â€” tracks known agents on ACN with trust levels, skills, endpoints, and interaction history

## Implementation Files

| File | Role |
|------|------|
| `lib/generator/social.js` | Generates `agent-card.json`, `acn-config.json`, and `contacts.json` seed |
| `lib/social/contacts.js` | Runtime CRUD for `social/contacts.json` + `social/contacts.jsonl` |
| `lib/social/acn-client.js` | Read-only ACN HTTP client: `fetchAgent`, `searchAgents`, `syncContacts`, `autoDiscover` |
| `lib/social/http.js` | Thin HTTP helper (GET/POST, no external deps) used by `acn-client.js` |
| `lib/remote/registrar.js` | ACN registration + auto-discover hook |

## CLI Commands

```bash
# Manage contacts
openpersona social list <slug>                           # List all contacts
openpersona social list <slug> --filter trust=community  # Filter by trust level
openpersona social add <slug> --from-acn <agent-id>     # Add from ACN lookup
openpersona social add <slug> --manual --file contact.json  # Add manually
openpersona social lookup <slug> <query>                 # Search local contacts
openpersona social search <slug> --skills <skill>        # Search ACN network
openpersona social sync <slug>                           # Refresh from ACN
openpersona social remove <slug> <agent-id>              # Remove a contact

# Register with ACN (auto-discover hook integrated)
openpersona acn-register [slug]
```

## Trust Levels

| Level | Meaning |
|-------|---------|
| `verified` | On-chain verified or manually reviewed |
| `community` | Has ACN history; not manually verified |
| `unverified` | Unknown/first-seen (default for auto-discover) |
